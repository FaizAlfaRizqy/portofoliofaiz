## Problems Encountered

### Hero content didn't overlap with the video.

Initially suspected:

- HTML structure
- `z-index`
- CSS positioning

Actual cause:

> The Vite development server was not running.

---

## Bug Fix

Started Vite using:

```bash
npm run dev
```

After that:

- CSS loaded correctly.
- Layout changes finally appeared in the browser.

---

## Things I Learned Today

- Laravel serves files from the `public/` directory.
- Blade files are stored inside `resources/views/`.
- Routes determine which view is displayed.
- Static assets belong inside `public/assets`.
- CSS in Laravel is compiled by Vite.
- If CSS changes don't appear, always check whether `npm run dev` is running before debugging the code.
