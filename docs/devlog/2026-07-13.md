# Biggest Bugs Today

## Navbar items stayed vertical

Cause:

Forgot to enable Flexbox.

Solution:

```css
display: flex;
```

---

## Hover affected the wrong element

Cause:

Incorrect CSS selector.

Solution:

Reviewed how CSS selectors actually read parent-child relationships.

---

## Wrong selector

Originally wrote:

```css
#navbar ul
```

Realized that the `ul` itself already owns the `id`.

Correct approach:

```css
#navbar
```

# Biggest Lessons Today

### 1.

Every CSS selector tells a story.

Instead of reading:

```css
.hero-content .btn:hover
```

as syntax,

I now read it as:

> "When the button inside Hero Content is hovered..."

---

### 2.

Always ask:

> "What is the reference point?"

This idea finally made CSS units easier to understand.

Examples:

- `px` → fixed value
- `%` → parent element
- `vh` → viewport height
- `vw` → viewport width

This feels very similar to UAV navigation:

- AGL (Above Ground Level)
- AMSL (Above Mean Sea Level)

The important question isn't the unit itself.

The important question is:

> "Relative to what?"

---

### 3.

Semantic HTML is about meaning.

Not appearance.
